===========================================
評価版（製品に組み込むにはカスタマイズ必須)
===========================================

Excelフォルダには、サンプルエクセルブック demo-book1　と
そのブックから呼び出している DLL、DLLが呼び出す音声ファイル
（wavファイル）が含まれます。

demo-book1 を開くと MxNet2サーバからライセンスの取得を試行、
成功すればブックが開きます。ライセンスを取得できなければ
終了します。例えば、30ライセンスがあれば、30台のコンピュータ
で同時に demo-book1を開くことができますが、31台目の
コンピュータでは開くことはできません。


USBキーでライセンス管理する方法との比較
~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~
- 多数のライセンスを付与するケースでは、ネットワークライセンス
の方が管理しやすくなります。少数ライセンスの場合、ドングルの方が
コスト、管理の両面で優位性があります。

- 物理的にドングルを接続できない環境でもネットワークライセンス
は運用可能。



Excel VBA で DLL を呼び出すことでネットワークを意識せずに
MxNet2サーバからライセンスを取得できます。demo-book1 の
VBAをご覧ください。


1. demo_book1 を開くときに CTRL+ALT を押していると、ライセンスチェックを
　しないでブックを開くことができます。


2.  mxribig64-n/mxribig-n.DLL と同じフォルダ内に usercode　という
　　ファイルがなければなりません。usercode ファイルを取得するには、
　　Webインタフェースの "MxNet2”タブのテストプログラムをダウンロードします。
　　zipファイル内の usercode があります。それをmxribig64-n/mxribig-n.DLLと
　　同じフォルダにコピーしてください。


2. AppSlot は demo_book1 の ThisWorkBook で指定しています。サンプルでは7に
　なっています。


	Const APPSLOT = 7


4. DLLと同じフォルダ内の mxnetapi.ini で MxNet2サーバの IP/Port　を指定してください。


5. 他のブックでライセンス管理を行うには demo_book1 内のコードを　ブックにインポート/コピー
　します。

	A,ThisWorkbookの最後の部分以外をコピー/貼付

	B. Workbook_Open に Call usb_check を追加
	
		Private Sub Workbook_Open()

			Call usb_check
			...
			...
		End Sub

	C. 標準モジュール２つをエクスポート/インポート

		APIDEF
		EXCEL_XTEA 

	D. License Check という名前のシートを追加
